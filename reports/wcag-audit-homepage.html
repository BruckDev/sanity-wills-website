1.4.10 Reflow AAFailSerious

At the Browser's 320px setting (effective 305px viewport), document width was 355px. The mobile menu button and main content extended beyond the right edge.

Evidence
button.inline-flex.h-11 at x=321, right=355; div.max-w-3xl right=347; scrollWidth: 355.
Fix
  • Remove minimum/fixed horizontal widths and ensure the mobile layout uses a container width no wider than 100%.

1.1.1 Non-text Content AFailModerate

The hero background image has no alt attribute; Next.js also reported this in the console.

Evidence
img.absolute.inset-0 (/images/home/hero-background.png).
Fix
  • Use a meaningful alternative if the image conveys content, otherwise set alt="" and keep it out of the accessible name.

1.3.1 Info and Relationships AFailModerate

The document outline jumps from the page h1 to service-card h3s; several level-2 headings have no accessible name.

Evidence
h1.max-w-5xl.text-5xl → h3; accessibility snapshot contains multiple heading [level=2] entries with no name.
Fix
  • Introduce descriptive h2 section headings before related h3 cards and remove empty headings.

1.4.3 Contrast (Minimum) AAFailModerate

The small Article and Perspective labels use accent green on white at 2.91:1, below the 4.5:1 minimum.

Evidence
div.flex.flex-wrap.items-center.gap-3.text-xs.font-semibold.uppercase.tracking-[0.2em].text-[color:var(--accent)] > span — foreground rgb(32,174,73), background white, 12px text.
Fix
  • Darken --accent for small text or use a darker semantic text token that reaches 4.5:1 on white.

2.4.6 Headings and Labels AAFailModerate

Multiple exposed level-2 headings have no accessible name, so they do not describe the following section.

Evidence
h2 — accessibility snapshot reported unnamed level-2 headings.
Fix
  • Populate each retained heading with concise visible text; replace empty structural headings with non-heading elements where no section exists.
